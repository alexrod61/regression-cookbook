12  Zero-Inflated Poisson Regression

When to Use and Not Use Zero-Inflated Poisson Regression

Zero-Inflated Poisson (ZIP) regression is useful when:

  • The outcome variable is discrete and count-type, taking values \(0, 1, 2, \dots\).
  • The data contain more zeros than a standard Poisson model would typically expect.
  • It is scientifically plausible that zeros may arise from more than one mechanism.
  • We want to model both whether an observation belongs to a certain-zero group and, separately, the expected count among observations that remain in the count-generating process.

However, ZIP regression should not be used automatically whenever zeros appear in a dataset. In particular:

  • If the outcome is binary, use the model introduced in Chapter 8.
  • If the outcome is a count but the data do not exhibit unusual zero inflation, start with the Classical Poisson model introduced in Chapter 10.
  • If the outcome is overdispersed for reasons not primarily driven by excess zeros, compare against the Negative Binomial model introduced in Chapter 11.
  • If there is no substantive reason to believe that a separate zero-generating process exists, a simpler count model is usually easier to interpret.

Learning Objectives

By the end of this chapter, you will be able to:

  • Explain why Zero-Inflated Poisson (ZIP) regression is useful for count outcomes with more zeros than a standard Poisson model would typically expect.
  • Describe the difference between structural zeros and ordinary sampling zeros in a count-data setting.
  • Identify the response variable and main explanatory variables in the insurance-claims example used throughout the chapter.
  • Use exploratory summaries and plots to assess whether a standard Poisson model may be inadequate.
  • Write the two components of a ZIP model: a zero-inflation model and a Poisson count model.
  • Fit and compare Classical Poisson or Logistic Regression with ZIP models using both R and Python.
  • Interpret coefficients from the count component and the zero-inflation component.

12.1 Introduction

In many applied regression problems, the outcome of interest is a count: the number of events, occurrences, or cases observed for an individual or unit over a fixed period of time. Examples include the number of hospital visits, the number of customer complaints, the number of defects on a manufactured item, or the number of insurance claims filed by a policyholder in a year.

A common starting point for this kind of outcome is Classical Poisson regression, introduced in Chapter 10, which is designed for nonnegative integer-valued responses and is part of the broader GLM framework (Nelder and Wedderburn 1972; Cameron and Trivedi 2013). However, in practice, some count datasets contain many more zeros than a simple Poisson model would be expected to generate. In those situations, a standard Poisson regression may fit poorly, especially near zero, and may fail to represent the data-generating process in a meaningful way (Lambert 1992; Cameron and Trivedi 2013).

This chapter introduces Zero-Inflated Poisson (ZIP) regression, a model designed for count data with excess zeros (Lambert 1992). ZIP regression is similar to a generalized linear model (GLM) because each part uses a familiar GLM-style link function (Nelder and Wedderburn 1972). More precisely, though, it is best understood as the combination of two linked regression components: a logistic-style model for whether an observation belongs to a zero-generating state, and a Poisson log-link model for the count-generating state. Thus, ZIP regression connects naturally to Chapter 8 and Chapter 10 respectively, but it is not just a single ordinary GLM.

Definition of Zero-Inflated Poisson

A Zero-Inflated Poisson (ZIP) model is a two-component model for a count response with excess zeros. One component models the probability that an observation belongs to a latent group that can only produce zeros. The other component models the expected count among observations that remain in the ordinary Poisson count-generating process.

Throughout the chapter, we use a synthetic insurance-claims dataset as a running example. In this setting, the response of interest is the number of claims filed by a policyholder during a fixed time window. This is a useful example because insurance data often contain a large proportion of zeros: many policyholders do not file any claims at all, while a smaller subset files one or more. That practical feature makes the insurance setting a natural context for introducing zero-inflated modelling.

12.2 Running Example and Data Context

To motivate ZIP regression, we use the synthetic insurance-claims data prepared for this chapter of the Regression Cookbook. The CSV is available from the cookbook GitHub repository: raw-data/zero_inflated_poisson.csv (Tai, Man-Yeung (Andy) 2026). We treat this file as the chapter’s teaching dataset rather than as an external real-world insurance portfolio.

Each row in the dataset represents one policyholder, and the main outcome records how many insurance claims that policyholder filed during the observation window. The response variable is claims, a nonnegative integer-valued count: \(0, 1, 2, \dots\)

The dataset contains the following variables.

A value of zero for the response means that the policyholder filed no claims during the observation period. Importantly, that zero can be interpreted in more than one way. In some cases, the policyholder may be effectively outside the claims-generating process for that period, or had such limited exposure that a positive claim count was structurally unlikely. In other cases, the policyholder may have been fully at risk of filing a claim but simply happened not to do so. This distinction is central to the two-part structure of the ZIP model, which will be discussed later in the data-modelling section.

For the sake of the problem, we will assume our primary objective is to determine which variables are relevant to predicting the number of claims, which increase the number in general, which decrease it, and whether we can use this relationships for predicting the outcome of any given policyholder.

12.3 Data Splitting

Following the data science workflow introduced in Section 1.4, we split the data before conducting EDA or fitting models. The training set is used for exploratory analysis, model fitting, and coefficient interpretation. The testing set is held out for later prediction and model assessment.

In this chapter, the split is created by shuffling the row order with seed 123 and then cutting the shuffled data at the 80% threshold for training. We intentionally do not stratify by has_claim; with a simple random shuffle, the sample itself is expected to carry the excess-zero structure into the training and testing sets.

12.4 Exploratory Data Analysis (EDA)

Before fitting any model, we examine whether the empirical distribution of claims in the training data is consistent with a simple Poisson pattern. The main EDA question is not merely “Are there zeros?” Count data often include zeros. The question is whether there are more zeros than the Poisson mean-variance structure would lead us to expect, and whether the covariates show patterns that make a two-component model plausible.

Because the split was performed before EDA, the numerical summaries in this section should be interpreted as training-set summaries. The exact values are the same in R and Python because both languages now use the canonical R split.

12.5 Preparing for Modelling

The data schema confirms that the response is a zero-inflated count and that the available regressors are either continuous, integer-valued, or binary. There are no multi-category nominal covariates in this dataset, so we do not need to create a baseline category plus a collection of dummy variables. The binary covariates are already coded as \(0/1\) indicators, and the continuous or integer-valued regressors can enter the model directly unless a later modelling choice calls for centering, scaling, or nonlinear transformations.

For policyholder \(i\), define the response as

\[Y_i = \texttt{claims}_i,\]

where \(Y_i\) is the number of claims filed during the observation window. The candidate covariates are indexed in the same order as the variable table above, excluding the response.

Using this notation, a compact covariate vector for the full set of candidate regressors is

\[\mathbf{x}_i = \left(1, x_{1,i}, x_{2,i}, x_{3,i}, x_{4,i}, x_{5,i}, x_{6,i}\right)\]

Where the leading \(1\) represents the intercept. In the ZIP model, this covariate vector can be used to build two linked regression components: a count component for the expected number of claims among policyholders in the count-generating process, and a zero-inflation component for the probability of belonging to the extra-zero group. Later sections can choose whether both components use the same covariates or whether one component uses a smaller, substantively motivated subset.

12.6 Comparing to Other Approaches

Before introducing the full Zero-Inflated Poisson (ZIP) model, it is useful to compare it with two simpler approaches that each captures only part of the modelling problem. The goal of this comparison is not only to fit two baselines, but also to see what each baseline reveals about the insurance-claims data.

First, we can turn the count response into a binary response. Define \(H_i = \mathbb{I}(Y_i > 0)\) where \(Y_i\) is the number of claims for policyholder \(i\), and \(\mathbb{I}\) is the indicator function, which outputs \(1\) in the case that \(Y_i>1\), and \(0\) in any other case. A Binary Logistic Regression model for has_claim then describes whether a policyholder filed at least one claim. This is directly connected to the large zero-versus-nonzero split observed in Figure 12.2 and Table 12.7, but it does not distinguish between one claim and several claims.

Second, we can keep the original count response and fit a Classical Poisson Regression model for claims. This model describes the expected number of claims directly, so it keeps the count scale used in Figure 12.4 and Table 12.13. However, it treats all observations as coming from a single count-generating process. In particular, it ties the zero rate to the same mean parameter that governs the positive counts.

These two models are not replacements for ZIP regression in this chapter. Instead, they give us useful comparison points. Logistic Regression focuses on the zero-versus-nonzero distinction. Classical Poisson Regression focuses on the count scale. ZIP regression combines these ideas by allowing a separate zero-generating component and a count-generating component.

12.6.1 Binary Logistic Regression for has_claim

Model definition

The Logistic Regression alternative uses the derived binary response in Equation 12.1.

\[H_i = \begin{cases} 1, & \text{if policyholder } i \text{ filed at least one claim}, \\ 0, & \text{if policyholder } i \text{ filed no claims}. \end{cases} \tag{12.1}\]

The probabilistic assumption is given in Equation 12.2, where \(\pi_i = P(H_i = 1 \mid \mathbf{x}_i)\) is the probability that policyholder \(i\) filed at least one claim.

\[H_i \mid \mathbf{x}_i \sim \operatorname{Bernoulli}(\pi_i) \tag{12.2}\]

The link function is the logit link in Equation 12.3.

\[\operatorname{logit}(\pi_i) = \log\left(\frac{\pi_i}{1 - \pi_i}\right) \tag{12.3}\]

Using the six covariates from the running example, the systematic component is shown in Equation 12.4.

\[\log\left(\frac{\pi_i}{1 - \pi_i}\right) = \alpha_0 + \alpha_1 x_{1,i} + \alpha_2 x_{2,i} + \alpha_3 x_{3,i} + \alpha_4 x_{4,i} + \alpha_5 x_{5,i} + \alpha_6 x_{6,i} \tag{12.4}\]

In Equation 12.4, there are \(7\) coefficients, \(\alpha_0, \alpha_1, \ldots, \alpha_6\), which change the log-odds of having at least one claim. The intercept \(\alpha_0\) is the baseline log-odds when all covariates are equal to zero, while \(\alpha_j\) for \(j \in \{1, 2, \ldots, 6\}\) represents the change in log-odds for a one-unit increase in the corresponding regressor \(x_{j,i}\), holding the other covariates fixed.

As in Chapter 8, this “one-unit increase” interpretation should be read according to the type of covariate. For a continuous covariate, such as driver_age or annual_mileage, \(\alpha_j\) describes the change in log-odds associated with increasing that covariate by one unit, holding the others fixed. For an integer-valued covariate, such as vehicle_age, the interpretation is similar, but the one-unit change corresponds to one additional year of vehicle age. For a binary indicator, such as male, urban_area, or prior_claims, \(\alpha_j\) compares the group coded \(1\) with the reference group coded \(0\). If a nominal categorical covariate with more than two categories were included, its coefficients would be interpreted relative to the omitted reference category, rather than as a literal one-unit increase.

In this formulation, the model describes the probability of at least one claim. It intentionally ignores the difference between one claim and multiple claims.

12.6.2 Classical Poisson Regression for claims

Model definition

The Classical Poisson Regression alternative keeps the original count response, defined in Equation 12.5.

\[Y_i = \text{number of claims filed by policyholder } i \tag{12.5}\]

The probabilistic assumption is shown in Equation 12.6, where \(\lambda_i = \mathbb{E}(Y_i \mid \mathbf{x}_i)\) is the expected number of claims for policyholder \(i\).

\[Y_i \mid \mathbf{x}_i \sim \operatorname{Poisson}(\lambda_i) \tag{12.6}\]

The link function is the log link in Equation 12.7.

\[g(\lambda_i) = \log(\lambda_i) \tag{12.7}\]

Using the same six covariates, the systematic component is given in Equation 12.8.

\[\log(\lambda_i) = \beta_0 + \beta_1 x_{1,i} + \beta_2 x_{2,i} + \beta_3 x_{3,i} + \beta_4 x_{4,i} + \beta_5 x_{5,i} + \beta_6 x_{6,i} \tag{12.8}\]

In Equation 12.8, there are \(7\) coefficients, \(\beta_0, \beta_1, \ldots, \beta_6\), which change the log-mean number of claims. The intercept \(\beta_0\) is the baseline log-mean when all covariates are equal to zero, while \(\beta_j\) for \(j \in \{1, 2, \ldots, 6\}\) represents the change in log-mean claims for a one-unit increase in the corresponding regressor \(x_{j,i}\), holding the other covariates fixed.

As in Chapter 10, this coefficient interpretation also depends on how the regressor is coded. For a continuous covariate, such as driver_age or annual_mileage, \(\beta_j\) describes the change in the log expected count for a one-unit increase in that covariate, holding the others fixed. For an integer-valued covariate, such as vehicle_age, the one-unit comparison corresponds to one additional year of vehicle age. For binary indicators, such as male, urban_area, or prior_claims, \(\beta_j\) compares the expected count for the group coded \(1\) against the reference group coded \(0\). If a nominal categorical covariate with more than two categories were included, its coefficients would compare each included category with the omitted reference category.

This model works directly on the count scale through \(\lambda_i\), but it also imposes the Poisson mean-variance relationship in Equation 12.9.

\[\mathbb{E}(Y_i \mid \mathbf{x}_i) = \operatorname{Var}(Y_i \mid \mathbf{x}_i) = \lambda_i \tag{12.9}\]

12.6.3 Why ZIP can be useful here

The two simpler approaches highlight the motivation for ZIP regression.

In the insurance-claims setting, the EDA suggested a large zero-claim group and a positive-count pattern among the remaining observations. Logistic Regression describes the first piece: who is likely to file at least one claim. Classical Poisson Regression describes the second piece: how expected claim counts change with the covariates, but it has to use the same mean structure to explain both the many zeros and the positive-count trend. ZIP regression is useful because it gives these two pieces separate but connected roles. One component models extra zeros, while the other models claim counts among observations that remain in the count-generating process.

12.7 Data Modelling

The previous section used Binary Logistic Regression and Classical Poisson Regression as baseline comparisons. We now define the model that this chapter is actually about: Zero-Inflated Poisson Regression. The goal of this section is to move from the empirical motivation to the formal modelling equations that will later be estimated in R and Python.

We first describe the model generically, without attaching the covariates to the insurance example. After that, we translate the notation back to the variables defined in Table 12.18.

12.7.1 Response distribution: a zero-inflated Poisson random variable

Let \(Y_i\) be a nonnegative count response for observational unit \(i\), where \(i = 1, 2, 3, \ldots, n\), and \(n\) is the number of observations. A Zero-Inflated Poisson model assumes that each observation can arise from one of two latent states:

  1. A structural-zero state, where the response is forced to be zero.
  2. A count-generating state, where the response follows an ordinary Poisson distribution and can still equal zero by chance.

To represent this idea, let’s define a latent indicator \(S_i\) which will work as an auxiliary variable to represent the presence of a structural zero. We have then that \(S_i = 1\) means that observation \(i\) belongs to the structural-zero state, and \(S_i = 0\) means that observation \(i\) belongs to the Poisson count-generating state. Let

\[ S_i \sim \operatorname{Bernoulli}(\pi_i), \tag{12.10}\]

where \(\pi_i = P(S_i = 1)\) is the probability that observation \(i\) belongs to the structural-zero state. Conditional on this latent state, the response is generated as follows:

\[ Y_i \mid S_i = \begin{cases} 0, & \text{if } S_i = 1, \\ \operatorname{Poisson}(\lambda_i), & \text{if } S_i = 0. \end{cases} \tag{12.11}\]

The parameter \(\lambda_i > 0\) is the Poisson mean in the count-generating state. The structural-zero state is important because it adds zeros that cannot be explained by the ordinary Poisson process alone. In other words, some zeros are structural zeros, while other zeros are ordinary sampling zeros generated by the Poisson distribution.

Using Equation 12.10 and Equation 12.11, the probability of observing zero can be built by separating the two latent states. The event \(Y_i = 0\) can happen in two ways:

  1. the observation belongs to the structural-zero state, or
  2. the observation belongs to the count-generating state and the Poisson random variable happens to equal zero.

The law of total probability gives

\[ \begin{aligned} P(Y_i = 0) &= P(Y_i = 0, S_i = 1) + P(Y_i = 0, S_i = 0) \\[6pt] &= \underbrace{P(S_i = 1)}_{\substack{\text{structural-zero} \\ \text{state}}} + \underbrace{P(S_i = 0)}_{\substack{\text{count-generating} \\ \text{state}}} \underbrace{P(Y_i = 0 \mid S_i = 0)}_{\substack{\text{ordinary Poisson} \\ \text{zero}}} \\[6pt] &= \underbrace{\pi_i}_{\substack{\text{extra-zero} \\ \text{probability}}} + \underbrace{(1 - \pi_i)}_{\substack{\text{probability of being} \\ \text{in the count state}}} \quad \cdot \quad \underbrace{\exp(-\lambda_i)}_{\substack{\text{Poisson probability} \\ \text{of zero}}} \\[6pt] &= \pi_i + (1 - \pi_i)\exp(-\lambda_i). \end{aligned} \tag{12.12}\]

The first term is the extra mass placed at zero by the structural-zero state. The second term is the ordinary Poisson zero probability, but it is weighted by \(1 - \pi_i\) because it can occur only among observations in the count-generating state.

For a positive count \(y_i = 1, 2, 3, \ldots\), the structural-zero state cannot produce the observed value. Therefore, the only contributing path is the count-generating state followed by a Poisson draw equal to \(y_i\):

\[ \begin{aligned} P(Y_i = y_i) &= P(Y_i = y_i, S_i = 1) + P(Y_i = y_i, S_i = 0) \\[6pt] &= \underbrace{P(S_i = 1)P(Y_i = y_i \mid S_i = 1)}_{\substack{=\,0 \\ \text{structural-zero state cannot} \\ \text{produce a positive count}}} + \underbrace{P(S_i = 0)}_{\substack{\text{count-generating} \\ \text{state}}} \underbrace{P(Y_i = y_i \mid S_i = 0)}_{\substack{\text{Poisson probability} \\ \text{of } y_i}} \\[6pt] &= 0 + \underbrace{(1 - \pi_i)}_{\substack{\text{probability of being} \\ \text{in the count state}}} \quad \cdot \quad \underbrace{\frac{\exp(-\lambda_i)\lambda_i^{y_i}}{y_i!}}_{\substack{\text{Poisson PMF} \\ \text{at } y_i}} \\[6pt] &= (1 - \pi_i)\frac{\exp(-\lambda_i)\lambda_i^{y_i}}{y_i!}, \qquad y_i = 1, 2, 3, \ldots \end{aligned} \tag{12.13}\]

Equations Equation 12.12 and Equation 12.13 define the Zero-Inflated Poisson probability model.

More formally, this distribution is represented as the following Probability Mass Function (PMF):

\[ P(Y_i=y_i)= \begin{cases} \pi_i+(1-\pi_i)\exp(-\lambda_i), & \text{if } y_i = 0 \\[6pt] (1-\pi_i)\dfrac{\exp(-\lambda_i) \lambda_i^{y_i}}{y_i!}, & \text{if } y_i = 1, 2, 3 \dots \end{cases} \tag{12.14}\]

This also shows why the ZIP model can produce more zeros than an ordinary Poisson model: the zero probability is not only \(\exp(-\lambda_i)\), but \(\pi_i + (1 - \pi_i)\exp(-\lambda_i)\).

The conditional mean under this model is

\[ \mathbb{E}(Y_i) = \sum_{y_i=0}^\infty y_i P(Y_i=y_i) = (1 - \pi_i)\lambda_i. \tag{12.15}\]

Thus, the expected count depends on both components. A larger \(\lambda_i\) increases the expected count among observations in the count-generating state, while a larger \(\pi_i\) lowers the marginal expected count by moving more observations into the structural-zero state.

Tip on deriving the expected value of a ZIP random variable

A convenient way to derive the expected value of a Zero-Inflated Poisson random variable is to start directly from its PMF and then notice that the zero term contributes nothing to the expectation.

Suppose that, for policyholder \(i\),

\[ P(Y_i = y_i) = \begin{cases} \pi_i + (1 - \pi_i)\exp(-\lambda_i), & y_i = 0, \\[6pt] (1 - \pi_i)\dfrac{\exp(-\lambda_i)\lambda_i^{y_i}}{y_i!}, & y_i = 1, 2, 3, \ldots \end{cases} \]

where \(\pi_i\) is the probability of belonging to the structural-zero state, and \(\lambda_i\) is the Poisson mean in the count-generating state.

The expectation is

\[ \mathbb{E}(Y_i) = \sum_{y_i = 0}^{\infty} y_i P(Y_i = y_i). \]

The first term of this sum corresponds to \(y_i = 0\). Since it is multiplied by zero, it drops out:

\[ \mathbb{E}(Y_i) = \sum_{y_i = 1}^{\infty} y_i (1 - \pi_i) \dfrac{\exp(-\lambda_i)\lambda_i^{y_i}}{y_i!}. \]

Now factor out the terms that do not depend on \(y_i\):

\[ \mathbb{E}(Y_i) = (1 - \pi_i)\exp(-\lambda_i) \sum_{y_i = 1}^{\infty} \dfrac{y_i\lambda_i^{y_i}}{y_i!}. \]

Since

\[ \dfrac{y_i}{y_i!} = \dfrac{1}{(y_i - 1)!}, \]

we can rewrite the sum as

\[ \mathbb{E}(Y_i) = (1 - \pi_i)\exp(-\lambda_i) \sum_{y_i = 1}^{\infty} \dfrac{\lambda_i^{y_i}}{(y_i - 1)!}. \]

Now change the index by letting \(p = y_i - 1\). When \(y_i = 1\), \(p = 0\), so

\[ \mathbb{E}(Y_i) = (1 - \pi_i)\exp(-\lambda_i) \sum_{p = 0}^{\infty} \dfrac{\lambda_i^{p + 1}}{p!}. \]

Factor out one \(\lambda_i\) from \(\lambda_i^{p+1}\):

\[ \mathbb{E}(Y_i \mid \mathbf{x}_i) = (1 - \pi_i)\exp(-\lambda_i) \sum_{p = 0}^{\infty} \dfrac{\lambda_i^{p}\lambda_i}{p!} = (1 - \pi_i)\exp(-\lambda_i)\lambda_i \sum_{p = 0}^{\infty} \dfrac{\lambda_i^p}{p!}. \]

Finally, recall the exponential McLaurin series expansion

\[ \exp(x) = \sum_{n = 0}^{\infty} \dfrac{x^n}{n!} \] where \(x\) is equivalent to \(\lambda_i\), and \(n\) is equivalent to \(p\).

Therefore,

\[ \mathbb{E}(Y_i) = (1 - \pi_i)\exp(-\lambda_i)\lambda_i\exp(\lambda_i) = (1 - \pi_i)\exp(-\lambda_i + \lambda_i) \cdot \lambda_i = (1 - \pi_i)\lambda_i. \]

So, for a ZIP model,

\[ \boxed{ \mathbb{E}(Y_i) = (1 - \pi_i)\lambda_i } \]

This result is intuitive: the count-generating Poisson process has mean \(\lambda_i\), but it is active only with probability \(1 - \pi_i\). The structural-zero group contributes no counts to the expectation.

12.7.2 Modelling the parameters with covariates

Now suppose that each observational unit has \(k\) candidate covariates. A compact covariate vector can be written as

\[ \mathbf{x}_i = (1, x_{1,i}, x_{2,i}, \ldots, x_{k,i})^\top, \tag{12.16}\]

The main implication of combining a structural-zero state with a count-generating state is that the model has two systematic components instead of one. One component describes the structural-zero probability \(\pi_i\) from Equation 12.10, and the other describes the Poisson mean \(\lambda_i\) from Equation 12.11. This is the main difference from the single-component GLM examples introduced earlier in the cookbook.

For this chapter, we use \(\alpha\) coefficients for the structural-zero component and \(\beta\) coefficients for the count component.

The structural-zero probability is modelled with a logit link:

\[ \operatorname{logit}(\pi_i) = \log\left(\frac{\pi_i}{1 - \pi_i}\right) = \alpha_0 + \alpha_1x_{1,i} + \alpha_2x_{2,i} + \cdots + \alpha_kx_{k,i}. \tag{12.17}\]

The fraction inside the logarithm,

\[ \frac{\pi_i}{1 - \pi_i}, \]

is an odds. In this ZIP setting, it compares the probability of belonging to the structural-zero state, \(\pi_i\), with the probability of not belonging to that state, \(1 - \pi_i\). In other words, the logit link models the log of the ratio

\[ \frac{\text{probability of structural-zero membership}}{\text{probability of count-generating membership}}. \]

The right-hand side of Equation 12.17 is the linear systematic component. It is called linear because the coefficients enter additively on the link-function scale: an intercept plus one coefficient-weighted contribution from each covariate. The model is not claiming that \(\pi_i\) itself is a linear function of the covariates. Instead, it claims that the log-odds of structural-zero membership are linear in the covariates.

Alternatively we can also define the \(\boldsymbol{\alpha}\) vector as

\[ \boldsymbol{\alpha} = (\alpha_0, \alpha_{1}, \alpha_{2}, \ldots, \alpha_{k})^\top, \tag{12.18}\]

and in matrix-vector product form, the same equation is

\[ \operatorname{logit}(\pi_i) = \mathbf{x}_i^\top \boldsymbol{\alpha}. \tag{12.19}\]

Equivalently,

\[ \pi_i = \frac{\exp(\mathbf{x}_i^\top \boldsymbol{\alpha})}{1 + \exp(\mathbf{x}_i^\top \boldsymbol{\alpha})}. \tag{12.20}\]

A positive \(\alpha_j\) means that larger values of \(x_{j,i}\) are associated with higher log-odds of belonging to the structural-zero state, holding the other covariates fixed. On the exponential scale, \(\exp(\alpha_j)\) is an odds ratio for structural-zero membership. The exact interpretation depends on the type of covariate:

  • If \(x_{j,i}\) is continuous, then \(\alpha_j\) is the change in the log-odds of structural-zero membership for a one-unit increase in that covariate, holding all other covariates fixed. In practice, the one-unit change should be chosen carefully. For example, a one-kilometre change in annual mileage may be too small to communicate clearly, so a later interpretation may use \(10{,}000\alpha_j\) and \(\exp(10{,}000\alpha_j)\) instead.
  • If \(x_{j,i}\) is an ordered discrete or integer-valued covariate, then \(\alpha_j\) is still interpreted as a one-unit increase, but the unit has the meaning of the variable. For example, if vehicle age is measured in years, then \(\alpha_j\) is the change in log-odds associated with one additional year of vehicle age, holding the other covariates fixed.
  • If \(x_{j,i}\) is a binary indicator, then \(\alpha_j\) compares the group coded \(1\) with the group coded \(0\). For example, if prior_claims is coded as \(1\) for prior claims and \(0\) for no prior claims, then \(\exp(\alpha_j)\) is the structural-zero odds ratio for policyholders with prior claims relative to policyholders without prior claims, holding the other covariates fixed.
  • If \(x_{j,i}\) represents a nominal categorical variable with more than two categories, the interpretation depends on the dummy or indicator coding. Each coefficient compares one category with the reference category, not with a one-unit numerical increase. For instance, if vehicle type had categories such as sedan, SUV, and truck, and sedan were the reference category, then the SUV coefficient would compare SUVs with sedans, holding the other covariates fixed.

The Poisson count mean is modelled with a log link:

\[ \log(\lambda_i) = \beta_0 + \beta_1x_{1,i} + \beta_2x_{2,i} + \cdots + \beta_mx_{m,i}. \tag{12.21}\]

The log link plays a similar role for the count-generating mean. The left-hand side, \(\log(\lambda_i)\), is the logarithm of the Poisson mean among observations that remain in the count-generating state. The right-hand side is another linear systematic component: an intercept plus coefficient-weighted covariate contributions. This means that the model is linear on the log-mean scale, while \(\lambda_i\) itself remains positive after applying the inverse link in Equation 12.24.

Alternatively we can also define the \(\boldsymbol{\beta}\) vector as

\[ \boldsymbol{\beta} = (\beta_0, \beta_{1}, \beta_{2}, \ldots, \beta_{k})^\top, \tag{12.22}\]

and in matrix-vector product form, the same equation is

\[ \log(\lambda_i) = \mathbf{x}_i^\top \boldsymbol{\beta}. \tag{12.23}\]

Equivalently,

\[ \lambda_i = \exp(\mathbf{x}_i^\top \boldsymbol{\beta}). \tag{12.24}\]

A positive \(\beta_j\) means that larger values of \(x_{j,i}\) are associated with a higher log mean in the Poisson count-generating state, holding the other covariates fixed. On the exponential scale, \(\exp(\beta_j)\) is a rate ratio for the Poisson mean among observations that remain in the count-generating state. As with the \(\alpha\) coefficients, the exact interpretation depends on the type of covariate:

  • If \(x_{j,i}\) is continuous, then \(\beta_j\) is the change in the log mean count for a one-unit increase in the covariate, holding the other covariates fixed. The rate ratio for a larger change \(c\) is \(\exp(c\beta_j)\).
  • If \(x_{j,i}\) is an ordered discrete or integer-valued covariate, then \(\beta_j\) is the change in the log mean count for one additional unit of that variable. For example, if vehicle age is measured in years, then \(\exp(\beta_j)\) is the fitted multiplicative change in the count-generating mean for one additional year of vehicle age.
  • If \(x_{j,i}\) is a binary indicator, then \(\exp(\beta_j)\) compares the fitted count-generating mean for the group coded \(1\) with the group coded \(0\), holding the other covariates fixed.
  • If \(x_{j,i}\) represents a nominal categorical variable with more than two categories, each dummy-variable coefficient compares that category with the reference category. The coefficient should not be read as a numerical one-unit increase through the category labels.

In the current insurance-claims dataset, the available categorical variables are binary indicators. However, this nominal-categorical distinction is important for future applications of ZIP regression where variables such as region, policy type, vehicle class, or occupation group may enter the model.

Putting both components together shows how the two link functions determine the marginal mean of the observed count response. The count component contributes \(\lambda_i = \exp(\mathbf{x}_i^\top\boldsymbol{\beta})\), while the zero-inflation component contributes the probability of not being in the structural-zero state, \(1 - \pi_i\). Therefore, using Equation 12.15, Equation 12.20, and Equation 12.24, the overall mean can be written as follows:

\[ \begin{aligned} \mathbb{E}(Y_i \mid \mathbf{x}_i) &= (1-\pi_i)\lambda_i \\[10pt] &= \bigg(1-\frac{\exp(\mathbf{x}_i^\top \boldsymbol{\alpha})}{1 + \exp(\mathbf{x}_i^\top \boldsymbol{\alpha})}\bigg)\cdot \exp(\mathbf{x}_i^\top \boldsymbol{\beta})\\[10pt] &=\bigg(\frac{1}{1+\exp(\mathbf{x}_i^\top \boldsymbol{\alpha})} \bigg)\cdot \exp(\mathbf{x}_i^\top \boldsymbol{\beta})\\[10pt] &=\frac{\exp(\mathbf{x}_i^\top \boldsymbol{\beta})}{1+\exp(\mathbf{x}_i^\top \boldsymbol{\alpha})}\\[10pt] &=\dfrac{\exp(\beta_0+\beta_1 x_{1,i} + \beta_2 x_{2,i} + \dots +\beta_k x_{k,i})}{1+\exp(\alpha_0+\alpha_1 x_{1,i} + \alpha_2 x_{2,i} + \dots +\alpha_k x_{k,i})} \end{aligned} \tag{12.25}\]

Heads-up on using the same covariates in both components!

The notation above uses the same covariate vector \(\mathbf{x}_i\) in both components for simplicity and for consistency with the running example. In practice, the zero-inflation component and the count component can use different covariate sets. For example, one component could use all candidate regressors, while the other uses a smaller subset selected for substantive reasons: nature of the problem, feature selection, reducing model complexity, etc.

12.7.3 Assumptions behind the ZIP formulation

The ZIP model is useful only when its two-component story is plausible. In this chapter, the main assumptions are:

  • Count response: The response \(Y_i\) is a nonnegative integer-valued count. This is the scale assumed by the ZIP PMF in Equation 12.14. In the running example, claims satisfies this requirement because it records \(0, 1, 2, \ldots\) claims.

  • Independent observations: Conditional on the covariates and model parameters, observations are treated as independent. In practical terms, the model is not explicitly accounting for repeated measurements on the same policyholder, household clustering, geographic clustering, or time-series dependence. If those structures exist, the basic ZIP model may need to be extended.

  • Two zero sources: It is substantively plausible that some zeros are structural zeros, while other zeros arise from the Poisson count-generating process. This is the distinction encoded by \(S_i\) in Equation 12.10 and Equation 12.11. Without a credible two-source explanation for zeros, the extra zero component may be difficult to justify.

  • Correct component interpretation: The zero-inflation component models \(\pi_i\), the probability of belonging to the structural-zero state. The count component models \(\lambda_i\), the Poisson mean among observations that remain in the count-generating state. These are not interchangeable. A covariate in the \(\alpha\) component changes the structural-zero mechanism; a covariate in the \(\beta\) component changes the count-generating mean.

  • Link-function structure: The logit link in Equation 12.17 is assumed to be a reasonable way to connect covariates to \(\pi_i\), and the log link in Equation 12.21 is assumed to be a reasonable way to connect covariates to \(\lambda_i\). This means the model is additive and linear on the link scales, not necessarily on the original probability or mean scales.

  • Poisson count-generating state: Conditional on being in the count-generating state, the Poisson mean-variance relationship is a reasonable starting point. The ZIP model adds extra zeros, but it does not by itself solve every kind of overdispersion in the positive-count process.

  • Enough information for two components: There are enough positive counts, enough zeros, and enough covariate variation to estimate both the \(\alpha\) and \(\beta\) components without severe instability. Because both components are estimated at the same time, sparse data or weakly separated groups can make the two sets of coefficients difficult to identify precisely.

If the count-generating part is still strongly overdispersed after accounting for excess zeros, then a Zero-Inflated Negative Binomial model may be a more appropriate extension. If there is no substantive reason to distinguish structural zeros from ordinary sampling zeros, then a simpler count model may be easier to justify and communicate.

Heads-up on severe instability in ZIP models

In this context, severe instability means that the two-component ZIP model does not have enough information to estimate the structural-zero and count-generating mechanisms reliably. Symptoms may include convergence warnings, extremely large coefficient estimates, very wide standard errors, fitted probabilities very close to \(0\) or \(1\), or coefficient estimates that change dramatically after small changes to the data or covariate set.

This matters because the ZIP model estimates both \(\boldsymbol{\alpha}\) and \(\boldsymbol{\beta}\). If the data contain many zeros but very few positive counts, the model may struggle to decide whether a covariate belongs mainly in the structural-zero story, the count-generating story, or both. In those cases, coefficient tables should be interpreted cautiously, and predicted probabilities, fitted means, sensitivity checks, or simpler comparison models may be more informative.

Tip on Bayesian extensions for limited data

The ZIP model has a naturally hierarchical interpretation: an observation first belongs either to a structural-zero state or to a count-generating state, and then the count-generating state follows a Poisson model. When the dataset is limited, this two-component structure can make the fitted coefficients and their standard errors sensitive to small changes in the data.

A Bayesian version of ZIP regression can sometimes help in this situation by placing prior distributions on the \(\alpha\) and \(\beta\) coefficients. If those priors are sufficiently informative, relevant to the problem context, and carefully justified, they can regularize unstable estimates and make the uncertainty in both components more explicit.

That said, a Bayesian ZIP model is not automatically better. Weak or poorly chosen priors may not solve the instability problem, and overly strong priors can dominate the data. Bayesian ZIP regression is therefore a useful extension to know about, especially when data are sparse, but exploring it in detail is outside the scope of this chapter and outside the main purpose of this cookbook.

12.7.4 Interpretation caveats

This can produce results that feel counter-intuitive at first. For example, a covariate could have a positive \(\alpha_j\) and a positive \(\beta_j\). In that case, higher values of the covariate are associated with higher odds of being in the structural-zero state, but also with a higher expected count among observations that are not structural zeros. These statements are not contradictory because they refer to different latent parts of the model.

An example could be a public-health outreach programme. Suppose the response is the number of emergency department visits in a year, and one covariate indicates whether a person is enrolled in an intensive care-management programme. Enrollment might have a positive zero-inflation coefficient if many enrolled people are stable, closely monitored, and therefore unlikely to visit the emergency department at all. At the same time, enrollment might have a positive count-component coefficient because, among the enrolled people who are not in the structural-zero group, many are high-need patients who can have repeated emergency visits. The same covariate can therefore point toward more zeros in one latent subgroup and larger counts in the remaining at-risk subgroup.

A concrete insurance example, using a covariate not present in the current dataset, is enrollment in a dash-cam-based safe-driving programme. Such a programme could be associated with higher odds of structural-zero membership if many enrolled drivers are cautious drivers who rarely file claims. However, among enrolled drivers who are still in the count-generating state, claim counts could be higher if the programme is especially attractive to high-exposure drivers, fleet drivers, or drivers who report minor incidents more consistently because the dash-cam system records them. In that case, the zero-inflation coefficient and the count-component coefficient could both be positive without implying a contradiction.

The marginal expected count in Equation 12.15 depends on both \(\pi_i\) and \(\lambda_i\). Therefore, a coefficient in one component should not automatically be interpreted as the total effect of a covariate on the observed count. For a continuous covariate that appears in both components, the marginal mean combines the two linear predictors through Equation 12.25, so the direction of the overall fitted mean can depend on both \(\alpha_j\) and \(\beta_j\), as well as on the current value of \(\pi_i\).

Tip on reading ZIP coefficients

A positive count-component coefficient and a positive zero-inflation coefficient do not mean the model is wrong. They may indicate a population where a covariate separates two kinds of observations: a subgroup increasingly likely to have no events at all, and a remaining at-risk subgroup with larger expected counts. This is one reason predicted probabilities, fitted means, and marginal effects are often more intuitive than coefficient tables alone.

12.7.5 ZIP model for the insurance-claims running example

We now return to the notation from Table 12.18. For policyholder \(i\), the response is \(Y_i = \texttt{claims}_i\), and the six candidate covariates are

\[ \mathbf{x}_i = \left(1, x_{1,i}, x_{2,i}, x_{3,i}, x_{4,i}, x_{5,i}, x_{6,i}\right)^\top, \tag{12.26}\]

where \(x_{1,i}\) is driver_age, \(x_{2,i}\) is annual_mileage, \(x_{3,i}\) is vehicle_age, \(x_{4,i}\) is male, \(x_{5,i}\) is urban_area, and \(x_{6,i}\) is prior_claims.

For the zero-inflation component, we write

\[ \log\left(\frac{\pi_i}{1 - \pi_i}\right) = \alpha_0 + \alpha_1x_{1,i} + \alpha_2x_{2,i} + \alpha_3x_{3,i} + \alpha_4x_{4,i} + \alpha_5x_{5,i} + \alpha_6x_{6,i}. \tag{12.27}\]

Here, \(\pi_i\) is the probability that policyholder \(i\) belongs to the structural-zero group. For example, \(\alpha_2\) describes how annual mileage is associated with the log-odds of being in that structural-zero group, holding the other covariates fixed.

For the count component, we write

\[ \log(\lambda_i) = \beta_0 + \beta_1x_{1,i} + \beta_2x_{2,i} + \beta_3x_{3,i} + \beta_4x_{4,i} + \beta_5x_{5,i} + \beta_6x_{6,i}. \tag{12.28}\]

Here, \(\lambda_i\) is the expected number of claims for policyholder \(i\) conditional on being in the count-generating state. For example, \(\beta_2\) describes how annual mileage is associated with the log mean claim count among policyholders who remain in the count-generating process, holding the other covariates fixed.

Together, Equation 12.27 and Equation 12.28 define the working ZIP model for the insurance-claims example. This specification uses the same six covariates in both components, which is useful for teaching because it lets us compare how the same regressor can behave differently in the structural-zero and count-generating parts of the model.

The combination of these two would look like this for the mean response, recalling Equation 12.25:

\[ \mathbb{E}(Y_i \mid x_{1,i}, x_{2,i}, \dots, x_{6,i}) = \dfrac{\exp(\beta_0+\beta_1 x_{1,i} + \dots + \beta_6 x_{6,i})}{1 + \exp(\alpha_0+\alpha_1 x_{1,i} + \dots + \alpha_6 x_{6,i})} \tag{12.29}\]
